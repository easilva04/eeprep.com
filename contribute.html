Why Contribute?

EE Prep is an open-source educational platform built by the community, for the community. By contributing, you can:

  • Share your electrical engineering knowledge with students and professionals worldwide
  • Help improve the quality and accuracy of our educational content
  • Build your professional portfolio and demonstrate your expertise
  • Connect with other passionate engineers and educators

How to Contribute

1. Fork the Repository

Create your own copy of the EE Prep repository by forking it to your GitHub account.

Fork Repository

2. Clone to Your Local Machine

Clone your forked repository to your local machine using Git:

git clone https://github.com/YOUR-USERNAME/eeprep.com.git
cd eeprep.com

3. Set Up Upstream Remote

Link your local repository to the main project:

git remote add upstream https://github.com/easilva04/eeprep.com.git

4. Create a New Branch

Create a new branch for your changes:

git checkout -b feature/your-feature-name

5. Make Your Changes

Edit, add, or improve content in your local repository. You can:

  • Fix errors in existing topics
  • Add new examples or practice problems
  • Create new topic pages using our template at Config/template.html
  • Improve site functionality

6. Commit and Push

Commit your changes and push them to your forked repository:

git add .
git commit -m "Brief description of your changes"
git push origin feature/your-feature-name

7. Create a Pull Request

Submit a pull request from your branch to the main EE Prep repository:

  • Go to your fork on GitHub
  • Click "New Pull Request"
  • Select your feature branch
  • Add a description of your changes
  • Submit the pull request

Content Management

Using the HTML Template

We use a standardized HTML template for all content pages to ensure consistency across the site:

  • Find the template at Config/template.html
  • Copy this template as your starting point for any new page
  • Replace all placeholder text (text in [square brackets])
  • Maintain the HTML structure and class names
  • Test your page in a browser before submitting

Managing Content with pages.json

The pages.json file serves as our content database and helps maintain site structure:

  • Find the file at Topics/pages.json
  • Use the quickAdd.newPage section to add new content
  • Fill in the required fields (title, categoryId, filename, etc.)
  • Follow the examples provided in the file

Contribution Guidelines

Content Standards

  • Use clear, concise language appropriate for engineering students
  • Define technical terms on first use
  • Break down difficult concepts into manageable parts
  • Include examples for abstract concepts
  • Use headers and lists for organization
  • Include relevant equations (use LaTeX format)
  • Add diagrams or images as needed
  • Properly reference external sources and respect copyright

Code Standards

  • Follow responsive design principles
  • Comment your code appropriately
  • Ensure cross-browser compatibility
  • Optimize for performance
  • Use semantic HTML for accessibility
  • Include alt text for images
  • Ensure keyboard navigation works
  • Maintain sufficient color contrast
