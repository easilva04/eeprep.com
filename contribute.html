Why Contribute?

EE Prep is an open-source educational platform built by the community, for the community. By contributing, you can:

  • Share your electrical engineering knowledge with students and professionals worldwide
  • Help improve the quality and accuracy of our educational content
  • Build your professional portfolio and demonstrate your expertise
  • Connect with other passionate engineers and educators

Our GitHub Repository

Repository Information

EE Prep is hosted on GitHub, making it easy for contributors to collaborate. Our repository contains all the HTML, CSS, JavaScript, and content files that make up the website.

Repository URL: https://github.com/eeprep/eeprep.github.io

How to Contribute

1. Fork the Repository

Create your own copy of the EE Prep repository by forking it to your GitHub account.

Fork Repository

2. Clone to Your Local Machine

Clone your forked repository to your local machine using Git:

git clone https://github.com/[your-username]/eeprep.github.io.git

3. Create a New Branch

Create a new branch for your changes:

git checkout -b feature/your-feature-name

4. Make Your Changes

Edit, add, or improve content in your local repository. You can:

  • Fix errors in existing topics
  • Add new examples or practice problems
  • Create new topic pages using our template
  • Improve site functionality

5. Commit and Push

Commit your changes and push them to your forked repository:

git add .
git commit -m "Brief description of your changes"
git push origin feature/your-feature-name

6. Create a Pull Request

Submit a pull request from your branch to the main EE Prep repository. Our maintainers will review your changes and provide feedback.

Contribution Guidelines

Content Standards

  • Ensure all content is technically accurate and well-explained
  • Use clear, concise language appropriate for engineering students
  • Include relevant examples, diagrams, or illustrations when helpful
  • Properly reference external sources and respect copyright
  • Follow our existing formatting and style conventions

Code Standards

  • Maintain clean, well-commented HTML, CSS, and JavaScript
  • Ensure code is accessible and works across different browsers
  • Keep performance in mind, optimizing where possible
  • Test your changes thoroughly before submitting

Getting Support

Need help with your contribution? Here are some resources:

  • Review our detailed contribution guide
  • Raise questions in the GitHub Discussions
  • Contact a maintainer via contact@eeprep.com

We welcome contributors of all experience levels, and our community is ready to help you get started!
